    orderingscope(iscope=itemscope) click to toggle source

    Provide an array consisting of

    -- a scope for fetching ordered items,
-- a key suitable for passing to .order()
-- (optionally) a key for fetching the sort value from the scope (without ordering)

    USES @org parameter, which indicates how to sort the iscope relation

    # File app/models/results_cache.rb, line 282
def orderingscope iscope=itemscope
  [ iscope,
  case org
    when :newest
      'created_at'
    when :updated
      'updated_at'
    else # :ratings, :popularity, :random, :viewed, :approved, :referent_id, :recipe_count, :feed_count, :definition_count
      nil
  end ]
end

    When taking a slice out of the (singular) itemscope, load the associated entities meanwhile NB This is not valid when the cache uses multiple scopesâ€“but that should be handled by cache_and_partition

    # File app/models/results_cache.rb, line 301
def scope_slice offset, limit
  iscope = itemscope
  iscope = iscope.includes(:entity) if %w{ rcprefs taggings }.include? iscope.model_name.collection
  oscope, sort_key = orderingscope iscope
  oscope = oscope.order("#{sort_key} #{@sort_direction || 'DESC'}") if sort_key
  oscope.offset(offset).limit(limit).to_a
end
